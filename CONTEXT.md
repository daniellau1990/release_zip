# zip_rar_tool

压缩/解压工具的领域模型。覆盖 Core 库和 GUI 前端共享的术语。

## Language

### 核心概念

**压缩包**:
一个 ZIP、RAR 或 7z 格式的归档文件，包含一个或多个压缩后的文件。
_Avoid_: 归档文件、档案、Archive（仅代码中使用）

**解压**:
将压缩包中的所有文件还原到指定目录。
_Avoid_: 提取、Extract（仅代码中使用）、解压缩

**压缩**:
将一个文件夹中的所有文件打包并压缩为一个压缩包。
_Avoid_: 打包、Compress（仅代码中使用）

**格式**:
压缩包的类型，由文件后缀决定。支持的格式：ZIP、RAR、7z。
_Avoid_: 类型、Format（仅代码中使用）

### GUI 概念

**Tab（标签页）**:
主窗口顶部的三个切换页：解压、压缩、查看。用户一次只能在一个 Tab 中操作。
_Avoid_: 面板、页面、选项卡

**解压 Tab**:
选择压缩包 → 指定输出目录 → 可选输入密码 → 点击解压 → 进度条完成。
_Avoid_: 提取页

**压缩 Tab**:
选择源文件夹 → 选择格式（ZIP 默认 / 7z 备选）→ 可选输入密码 → 点击压缩 → 进度条完成。
_Avoid_: 打包页

**查看 Tab**:
选择压缩包 → 以表格显示文件列表（文件名、大小）。纯预览，无解压操作。
_Avoid_: 浏览、预览

**解压到**:
UI 上输出目录的标签文字。默认值为压缩包所在目录。
_Avoid_: 输出目录（仅代码 `output_dir` 使用）、目标路径

**密码（可选）**:
解压或压缩时可选的加密密码。三种格式（ZIP/RAR/7z）均支持。
_Avoid_: 加密密码、Password（仅代码中使用）

### 内部概念（用户不可见）

**progress_callback（进度回调）**:
`core.extract()` / `core.compress()` 的可选参数，签名为 `(current: int, total: int)`。Backend 在每处理完一个文件时调用，GUI 收到后更新进度条。CLI 不使用此参数。
_Avoid_: 进度通知、进度事件

**直接覆盖**:
解压或压缩时，若目标路径已存在同名文件，静默覆盖，不弹出确认对话框。
_Avoid_: 强制覆盖、覆盖模式
